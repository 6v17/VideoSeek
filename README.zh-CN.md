# VideoSeek 中文说明

`VideoSeek` 是一个基于 `PySide6 + ONNX Runtime + FAISS + FFmpeg` 的桌面语义视频检索工具。

## 功能概览

- 本地视频库：支持文本/图片检索、结果预览。
- 网络库：支持从在线视频链接构建向量库并直接检索。
- 向量管理：支持查看本地向量路径、查看网络链接明细、导入/导出网络库包。

## 安装与启动

1. 安装依赖：

```bash
pip install onnxruntime-gpu opencv-python PySide6 faiss-cpu numpy pillow ftfy regex yt-dlp
```

2. 启动：

```bash
python main.py
```

## 关键参数说明

配置文件：`config.json`

- `fps`：
  统一抽帧频率（本地索引 + 网络库构建共用）。
  - `1.0` = 每秒 1 帧
  - `0.5` = 每 2 秒 1 帧
  - `2.0` = 每秒 2 帧
- `remote_max_frames`（高级）：
  网络库单视频抽帧保护上限。通常无需修改。
  当视频过长且 `fps` 过高时，系统会自动降采样，避免构建过慢或内存占用过高。

## 网络库页面说明

- 构建模式：
  - `先下载再构建`：兼容性更高
  - `直接流构建`：速度更快，但对站点支持要求更高
- 页面按钮：
  - `查看链接列表`：查看已入库链接与时间范围
  - `打开下载目录`：快速打开缓存目录
- 进度显示：
  构建阶段会显示分段进度（解析/下载 -> 抽帧 -> 生成向量 -> 合并去重 -> 建索引）以及成功/失败/跳过摘要。

## 常见问题

1. 为什么结束时间常见到 299 秒？

- 老版本默认上限较小会导致采样主要覆盖前段。
- 当前版本已改进为按时长自适应采样并保留上限保护。
- 旧数据需要重建后才会反映新策略。

2. 同一个链接为什么会重复处理？

- 新版本已增强前置去重（在下载/抽帧前预判重复）。
- 若历史数据混杂旧格式，建议重建一次网络库以统一去重基准。

3. 为什么某些网站报 `Unsupported URL` 或 `Fresh cookies needed`？

- 很多是站点限制，不是本项目逻辑错误。
- 搜索页/频道页通常不支持，应使用具体视频页链接。
- 部分站点（如部分抖音页面）可能需要 cookies 才能被 `yt-dlp` 解析。

## 路径位置（默认）

- 网络索引：`data/remote/remote_index.faiss`
- 网络向量：`data/remote/remote_vectors.npy`
- 网络构建缓存：`data/remote_build_cache`
- 链接检索缓存：`data/link_cache`
